전송계층 - 데이터구조(세그먼트), 프로세서와 프로세서 간의 연결, 연결형(TCP)과 비연결형(UDP)
	게이트웨이
세션계층 - 대화채널들(오디오, 비디오, 텍스트 등)을 규정
	게이트웨이
표현계층 - 데이터 압축, 부호화, 복호화
	게이트웨이
응용계층 - 사용자 인터페이스, 응용 서비스 지원
	게이트웨이

TCP/IP 프로토콜
Network acess - MAC(48bit)_앞24bit는 제조업체 뒤24bit는 생산번호, Ethernet, Cable
	물리적주소(MAC 48비트) 사용
	프레임(2계층_MAC, HDLC등)
Internet - IP(번호), ICMP(번호), ARP(번호해석), RARP(번호역해석), RIP_OSPF_IGMP(경로설정)
	논리적주소(IP_32 또는 128비트) 사용
	패킷(IP패킷)
Transport - TCP(연결형_세그먼트), UDP(비연결형_데이터그램)
	포트번호(16비트) 사용
Application - TCP_(HTTP(웹), FTP(웹하드), TELNET(원격접속), SMTP(전자우편))
	, UDP_(DNS, TFTP, SNMP(망관리)
	특정 응용 서비스 엑세스(URL 주소, 메일 주소등)
	데이터형태_메시지

IPv4 주소
8비트씩 10진수로 구분하여 표현
32비트 - 네트워크 주소 부분과 호스트 주소 부분으로 구분
class A_1~127(50%)_8비트가 네트웍주소_16777214개 호스트 사용가능
class B_128~191(25%)_16비트가 네트웍주소_65534개
class C_192~223(12.5%)_24비트가 네트웍주소_254개
class D_224~239_멀티캐스트 주소로 사용
class E_240~255_미래를 위한 예비

32비트가 모두 0인 주소
0.0.0.0/32
호스트가 자신의 주소를 모르는 경우 
제한된 브로트캐스트 주소로 설정

32비트가 모두 1인 주소

네트워크 자체주소
